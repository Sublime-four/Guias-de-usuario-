FBZ Rappi Connector v18

Integración completa entre Odoo 18 y la plataforma Rappi. Recibe pedidos automáticamente, sincroniza productos y gestiona todo desde el Punto de Venta y la Cocina (KDS).

Versión 18.0.1.0

Categoría Ventas / Integración
Licencia Propietaria
1 Descripción General

Este módulo conecta tu sistema Odoo con la plataforma de delivery Rappi. Permite recibir pedidos, sincronizar tu catálogo de productos y gestionar todo el ciclo de vida de las órdenes sin salir de Odoo.

¿Qué es Odoo?
Es un software para administrar negocios: ventas, inventario, punto de venta, contabilidad y más. Todo desde un navegador web.
¿Qué es Rappi?
Es una plataforma de delivery donde los clientes piden comida y productos. Este módulo conecta tu tienda de Rappi con Odoo.
¿Qué hace este módulo?
Trae pedidos de Rappi a Odoo automáticamente, los envía al POS y a la cocina, y permite sincronizar tu menú.
¿Para quién es?
Restaurantes y negocios de comida que usan Rappi como canal de venta y quieren gestionar todo desde un solo lugar.
Funcionalidades principales
Funcionalidad Descripción
Pull Orders Descarga los pedidos de Rappi y crea automáticamente órdenes en Ventas y en el Punto de Venta (POS).
Pull Products Descarga todos los productos del menú de tu tienda en Rappi y los crea en Odoo si no existen.
Push Products Envía los productos de Odoo hacia Rappi para actualizar tu menú (filtra solo los que están listos para venta).
Webhook Recibe notificaciones en tiempo real desde Rappi cuando llega un nuevo pedido, sin necesidad de consultar manualmente.
Kitchen Display (KDS) Los pedidos de Rappi aparecen en la Pantalla de Cocina. Al terminar de preparar, la orden se cierra y paga automáticamente.
Tareas Automáticas El módulo renueva tokens y descarga pedidos automáticamente mediante tareas programadas (cron).

3 Instalación del Módulo

Sigue estos pasos para instalar el conector de Rappi en tu sistema Odoo:

  1. Inicia sesión en tu sistema Odoo con un usuario Administrador.
  2. Ve al menú principal y haz clic en Aplicaciones (icono de cuadrícula en la barra superior).
  3. En la barra de búsqueda, escribe FBZ Rappi Connector o rappi.
  4. Localiza el módulo "FBZ Rappi Connector v18" en los resultados. Verás el autor Coondev.
  5. Haz clic en el botón Instalar (o Activar si ya fue descargado).
  6. Espera a que Odoo termine la instalación. Aparecerá un nuevo menú llamado "Rappi" en la barra superior.
⚠️
Importante: Si no ves el módulo en la lista, asegúrate de que la carpeta fbz_rappi_connector_v18 esté dentro del directorio de addons de Odoo y que hayas actualizado la lista de aplicaciones (Aplicaciones → Actualizar lista de aplicaciones).
4 Configurar la Conexión con Rappi

Esta es la parte más importante. Aquí conectas tu Odoo con tu tienda de Rappi.

  1. En la barra superior, haz clic en el menú Rappi.
  2. Haz clic en Configuration (Configuración). Se abrirá la lista de conexiones.
  3. Haz clic en Nuevo para crear una configuración nueva.
  4. Rellena los campos de Conexión API:
    • Entorno: Selecciona Sandbox para pruebas o Production para tu tienda real.
    • Base URL: Se completa automáticamente según el entorno. No la cambies salvo que Rappi te indique otra.
    • Audience: Se completa automáticamente. Es la URL del API de Rappi.
    • Token URL: Se completa automáticamente. Es la URL de Auth0 para obtener tokens.
    • Store Name: El nombre de tu tienda/restaurante (ejemplo: "Mi Restaurante Centro").
    • Store ID: El identificador numérico que te dio Rappi.
    • Client ID: El identificador de tu aplicación del portal de Rappi.
    • Client Secret: La clave secreta del portal de Rappi.
    • Webhook Secret: (Opcional) Clave para verificar notificaciones automáticas.
  5. Haz clic en el botón Refresh Token para obtener tu primer token de autenticación.
  6. Haz clic en Test Connection para verificar que todo funciona correctamente. Deberías ver un mensaje "Conexión exitosa ✅".
  7. Haz clic en Guardar (icono de disquete o nube arriba a la izquierda).

5 Configurar el Punto de Venta (POS)

Para que los pedidos de Rappi lleguen directamente a tu Punto de Venta y a la cocina, configura estos campos adicionales.

  1. En la pantalla de configuración de Rappi (la misma del paso anterior), busca la sección "Routing & Pricing".
  2. Configura los siguientes campos:
    • Target POS: Selecciona el Punto de Venta donde aparecerán los pedidos de Rappi (ejemplo: "Restaurante Principal").
    • Método de pago Rappi: Selecciona o crea un método de pago llamado "Rappi" para que las órdenes se cierren automáticamente con ese método.
    • Equipo de Ventas: (Opcional) Equipo de ventas al que se asignarán las órdenes de venta.
    • Lista de Precios: (Opcional) Si tienes precios especiales para Rappi, selecciona la tarifa correspondiente.
    • Ubicación de Stock: (Opcional) Almacén/ubicación de donde se descontará el inventario.
  3. Haz clic en Guardar.
🔴
Obligatorio: Para que las órdenes se cierren y paguen automáticamente desde la cocina, debes configurar el campo "Método de pago Rappi". Sin este campo, el sistema no podrá marcar las órdenes como pagadas.
📌
Requisito del POS: El Punto de Venta seleccionado debe tener una sesión abierta para poder recibir pedidos. Si la sesión está cerrada, el módulo mostrará un error indicándote que la abras.
6 Recibir Pedidos de Rappi (Pull Orders)

Esta función descarga los pedidos de tu tienda de Rappi y los crea automáticamente en Odoo.

Método manual
  1. Ve a Rappi → Configuration y abre tu configuración.
  2. Haz clic en el botón "Pull Orders".
  3. Espera unos segundos. Verás una notificación indicando cuántos pedidos se procesaron.
¿Qué ocurre internamente?

Al hacer Pull Orders, el módulo:

  1. Se conecta a la API de Rappi y descarga todos los pedidos disponibles.
  2. Por cada pedido, crea o actualiza un registro rappi.order con el payload completo.
  3. Crea automáticamente una Orden de Venta (sale.order) con las líneas de productos.
  4. Si tienes un POS configurado, también crea una orden en el Punto de Venta con mesa asignada en el piso "Rappi".
  5. Envía la orden a la Pantalla de Cocina (KDS) para que el equipo de preparación la vea.
  6. Si el producto no existe en Odoo, lo crea automáticamente como producto de tipo servicio.
💡
Automático: No necesitas hacer esto manualmente. El sistema descarga pedidos cada 5 minutos de forma automática. Ver la sección de Tareas Automáticas.
7 Traer Productos de Rappi (Pull Products)

Esta función descarga todos los productos del menú actual de tu tienda en Rappi y los crea en Odoo si no existen.

  1. Ve a Rappi → Configuration y abre tu configuración.
  2. Haz clic en el botón "Pull Product".
  3. El sistema se conecta a Rappi y consulta el menú completo de tu tienda (endpoint /store/{store_id}/menu/current).
  4. Por cada producto del menú de Rappi:
    • Busca si ya existe en Odoo por su SKU (código interno) o por su Rappi SKU.
    • Si no existe, lo crea como producto almacenable (storable) con el nombre, precio, SKU y Rappi Item ID.
    • Si ya existe, lo omite (no lo duplica).
  5. Verás una notificación indicando cuántos productos nuevos se crearon.
📌
Sin filtros: El Pull Products trae todos los productos del menú de Rappi, sin excepción. Si un producto ya existe en Odoo (mismo SKU), simplemente se ignora para evitar duplicados.

8 Enviar Productos a Rappi (Push Products)

Esta función envía tu catálogo de productos desde Odoo hacia Rappi para actualizar el menú de tu tienda. A diferencia del Pull, aquí sí se aplican filtros para enviar solo productos listos.

  1. Ve a Rappi → Configuration y abre tu configuración.
  2. Haz clic en el botón "Push Product".
  3. El sistema busca productos que cumplan todos estos criterios:
    • Puede venderse (sale_ok = true) → El checkbox "Puede venderse" debe estar activado en la ficha del producto.
    • Disponible en POS (available_in_pos = true) → El producto debe estar habilitado para el Punto de Venta.
    • Precio mayor a 0 → No se envían productos sin precio.
    • Tiene SKU → El producto debe tener un Rappi SKU o una Referencia Interna (código).
  4. Los productos se agrupan por categoría para estructurar el menú en Rappi.
  5. Se envían al endpoint POST /menu de Rappi.
  6. Verás una notificación indicando cuántos productos se enviaron (y cuántos se omitieron si no cumplían los requisitos).
⚠️
Requisitos mínimos de Rappi:
  • El nombre del producto debe tener mínimo 3 caracteres.
  • La descripción de venta debe tener mínimo 3 caracteres.
  • Los productos que no cumplan se omiten automáticamente.

9 Webhook — Recepción Automática en Tiempo Real

El webhook permite que Rappi notifique a tu Odoo instantáneamente cuando llega un nuevo pedido, sin esperar al cron de 5 minutos.

Configurar el Webhook
  1. En el portal de desarrolladores de Rappi, busca la sección de Webhooks.
  2. Configura la URL del webhook con la dirección de tu servidor Odoo:
    URL: https://tu-dominio.com/rappi/webhook
    Método: POST
    Content-Type: application/json
  3. (Opcional) Si Rappi te proporciona un Webhook Secret, ingrésalo en el campo correspondiente de la configuración de Rappi en Odoo para validar la firma de cada mensaje.
¿Cómo funciona?
  1. Un cliente hace un pedido en Rappi.
  2. Rappi envía un mensaje (JSON) a https://tu-dominio.com/rappi/webhook.
  3. El módulo identifica la tienda por el storeId del mensaje.
  4. Si hay Webhook Secret, verifica la firma HMAC-SHA256 para asegurar que es auténtico.
  5. Procesa el pedido igual que si hubieras hecho "Pull Orders" manualmente.
📌
Eventos soportados: El webhook responde a eventos de tipo PING / HEALTH_CHECK (para verificar que el servidor está activo) y a eventos que contengan datos de order u orders.

Detalles del flujo POS

Cuando el módulo crea una orden en el POS, hace lo siguiente:

  1. Busca (o crea) un piso llamado "Rappi" en el POS configurado.
  2. Crea una mesa dentro de ese piso con el nombre/número del pedido de Rappi como identificador.
  3. Crea la orden POS asignada a esa mesa, con estado "borrador" (draft).
  4. Busca o crea un cliente con los datos que envía Rappi (nombre, teléfono, email, documento).
  5. Agrega las líneas de producto según los ítems del pedido. Si un producto no existe, lo crea automáticamente.
  6. Si hay descuento, agrega una línea negativa con producto "RAPPI-DISCOUNT".
  7. Envía la orden a la Pantalla de Preparación (KDS) para que la cocina la vea.
Estados de las órdenes en el rastreo

Verde completed, delivered, finished, done, paid El pedido fue completado y entregado.
Rojo cancelled, canceled, rejected El pedido fue cancelado o rechazado.
11 Pantalla de Cocina (Kitchen Display / KDS)

Los pedidos de Rappi aparecen automáticamente en la pantalla de cocina. Cuando el equipo marca el pedido como terminado, la orden POS se cierra y se paga automáticamente.

¿Cómo funciona el auto-cierre?
  1. El pedido de Rappi llega al sistema y aparece en la Pantalla de Preparación.
  2. El cocinero prepara el pedido y pulsa "Done" en la última etapa del KDS.
  3. El módulo intercepta esa acción y busca la orden POS vinculada.
  4. Elimina cualquier pago previo fantasma (si los hubiera).
  5. Crea automáticamente un pago por el total usando el Método de pago Rappi configurado.
  6. Marca la orden como pagada.
  7. Deja un mensaje de auditoría: "Orden cerrada automáticamente desde la Kitchen Display (Done)."
⚠️
Requisito: Para que el auto-cierre funcione, debes tener configurado:
  • Un Método de pago Rappi en la configuración del módulo.
  • Una Pantalla de Preparación vinculada al POS destino (se configura en Punto de Venta → Configuración → Pantallas de preparación).
💡
Limpieza automática: Cuando una orden Rappi se marca como pagada/completada, el módulo elimina automáticamente las órdenes "fantasma" (borradores con nota "Rappi #...") que pudieran haberse creado en la misma mesa, evitando duplicados.

15 Preguntas Frecuentes
¿El módulo soporta varias tiendas de Rappi?

Sí. Puedes crear una configuración (rappi.settings) por cada tienda. Cada una tendrá su propio Store ID, credenciales y POS destino.

¿Qué pasa si el POS no tiene sesión abierta?

La orden de venta (sale.order) se crea sin problema, pero la orden POS no se podrá crear. Verás un aviso indicando que debes abrir la sesión del POS.

¿Los productos se duplican si hago Pull varias veces?

No. El sistema busca por SKU antes de crear. Si el producto ya existe (mismo código interno o Rappi SKU), se omite.

¿Puedo usar el módulo solo para ventas (sin POS)?

Sí. Si no configuras un POS destino, el módulo solo creará órdenes de venta (sale.order). No intentará crear órdenes POS ni enviar a cocina.

¿Cómo veo solo las órdenes de Rappi en el POS?

El módulo agrega un menú "Órdenes Rappi" dentro del módulo de Punto de Venta. Allí puedes ver todas las órdenes POS que tienen la bandera is_rappi = true, con el tiempo transcurrido desde su creación.

¿Qué pasa con los descuentos de Rappi?

Si el pedido incluye un descuento (discountTotal o discount), el módulo crea una línea adicional con un producto llamado "Rappi Discount" (código: RAPPI-DISCOUNT) con monto negativo. Así el total de la orden refleja el descuento correctamente.

¿Qué navegadores soporta la pantalla de cocina (KDS)?

Cualquier navegador moderno: Chrome, Firefox, Edge, Safari. La pantalla de cocina es una página web dentro de Odoo, así que puedes abrirla en una tablet o un monitor dedicado.

¿El token se renueva solo?

Sí. El cron "Rappi: Refresh Token" se ejecuta cada 30 minutos automáticamente. También puedes renovarlo manualmente con el botón "Refresh Token" en la configuración.
